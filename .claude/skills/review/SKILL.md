# /review — 週次振り返り分析

1週間の予想結果を集約分析し、「なぜ外れたか」を構造的に特定する。

## 前提条件

- 対象週の `/collect` が完了していること（土日両方が望ましい）
- `data/results/` に結果ファイルと_review.jsonが存在すること

## 実行手順

### 1. 対象データの確認

```bash
# 対象週の結果ファイル一覧
ls data/results/ | grep "YYYY-MM-DD" | grep -v review | wc -l
ls data/results/ | grep "YYYY-MM-DD" | grep review | wc -l
```

### 2. スクリプト実行（存在する場合）

```bash
python3 scripts/run_review.py --week YYYY-WNN
```

スクリプトが未作成の場合は、以下の手動分析を行う。

### 3. 手動分析（スクリプト未作成時）

#### 3a. サマリー集計

対象週の全prediction + resultを読み込み、以下を集計:

- 全レース数 / BET数 / PASS数
- 総投資額 / 総回収額 / 収支 / ROI
- モデル1位の1着率（top1_hit_rate）
- モデルtop3と実際top3の重複率（top3_overlap_rate）

#### 3b. ファクター誤差分析

BETレースで結果が出ているものについて:
1. 各レースのprediction JSON（rankingsにfactorsがある場合）を読み込む
2. gap >= 3（モデル順位と実着順の差が3以上）の馬をリスト
3. その馬のファクタースコアを確認し、どのファクターが過大/過小評価されていたか特定
4. 全レースで集約し、系統的に偏っているファクターを報告

※ factorsフィールドがない古い予想の場合、course_info（距離・馬場）とstyle（脚質）からの推測分析に切り替える

#### 3c. パターン検出

外れたレースを以下でグルーピングし、再発パターンを探す:
- 馬場（芝/ダート）× 距離カテゴリ（短距離/マイル/中距離/長距離）
- 脚質（逃げ・先行の過大評価パターン等）
- 自信度（A/Bの的中率差）
- レースクラス（新馬・未勝利 vs オープン）

同カテゴリで3件以上のエラーがあり、エラー率60%超の場合 → pattern_signalとして記録

#### 3d. 大外し分析

gap >= 4 の馬について:
- どういう馬か（距離、脚質、騎手、人気）
- モデルが見落とした要素は何か
- 既存の仮説（knowledge/hypotheses.md）で説明できるか

#### 3e. 既存仮説へのエビデンス

knowledge/hypotheses.md（またはdata/hypotheses.json）の各仮説の条件に対し、今週のレースでマッチするものがあればsupports/refutesを判定

### 4. 出力

#### 4a. JSON出力（スクリプト使用時）
`data/reviews/YYYY-WNN_weekly_review.json` に保存

#### 4b. 報告フォーマット

```
## 週次レビュー: YYYY-WNN (MM/DD - MM/DD)

### サマリー
- 分析: NN レース（BET: N / PASS: M）
- 収支: ¥-XX,XXX（ROI: XX.X%）
- top1的中率: XX.X% / top3重複率: XX.X%

### ファクター傾向
- pace_advantage: 過大評価傾向（7レース中5レースで先行馬を高く付けすぎ）
- jockey_score: 概ね正確

### 検出パターン
1. **ダ短距離の差し馬過小評価** (4/5レース, 信頼度80%)
   → 仮説候補: ダ1200m以下でも差しが届くケースが増えている

### 大外し
- XX特別: モデル8位→実際1着（ゴールドシップ産駒・重馬場巧者を見落とし）

### 仮説エビデンス
- H-001: +1サンプル（支持）→ 通算 2/30
- H-003: +2サンプル（1支持, 1反証）→ 通算 2/30
```

### 5. Claudeの役割

数値の集計はPythonまたは手動計算。以下はClaudeが推論で担当:
- 「なぜそのパターンが起きたか」の解釈
- 出走取消・天候急変など、数値に現れない外部要因の指摘
- 新しい仮説候補の文章化
- 来週への具体的な改善示唆
