# /predict — 予想実行

指定日の全レースに対してスコアリングとBET/PASS判定を実行する。

## 前提条件

- レース当日の午前中（発走前）に実行する
- netkeiba.comにアクセス可能であること

## 実行手順

### 1. 引数の解釈

- 引数なし → 今日の全開催
- `--venue 東京` → 東京開催のみ
- `--date 20260426` → 日付指定（**YYYYMMDD形式**、ハイフンなし）
- `--official-only` → 公式モデルのみ（実験モデルをスキップ）

### 2. 予想実行

```bash
python3 scripts/run_predict.py [引数]
```

### 3. 結果確認

スクリプト完了後、以下を確認して報告する:

```bash
# 生成された予想ファイルを確認
ls -la data/predictions/ | grep "$(date +%Y-%m-%d)"
```

### 4. 報告フォーマット

```
## 予想完了: YYYY-MM-DD [会場名]

| # | レース名 | 判定 | 自信度 | 本命 | EV |
|---|----------|------|--------|------|-----|
| 1R | 3歳未勝利 | PASS | D | - | - |
| 5R | 葛飾特別 | BET | B | 7.レインボー | 1.45 |

**BET: N件 / PASS: M件**
**投資予定額: ¥XX,XXX**
```

### 5. 注意事項

- delay引数はデフォルト1.0秒。短くするとnetkeiba.comからブロックされる
- EV > 10 の馬は自動でPASS（オッズデータの異常）
- 新馬戦・未勝利戦はデータ不足でPASSになりやすい（正常動作）
